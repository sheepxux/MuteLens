# GISTER 人工标注指南 (Ground Truth Labeling Guide)

## 标注目标

在 `ground_truth_samples.csv` 的 `label` 列填写：
- **`1`** = 高价值，保留，推送给用户
- **`0`** = 低价值，过滤掉

---

## 判断标准（可操作定义）

### 打 `1` 的条件（满足任意一条即可）

| 条件 | 说明 | 示例 |
|------|------|------|
| **独到见解** | 作者有自己的观点，不只是转述事实 | "我认为 LLM 的真正瓶颈不是算力而是..." |
| **一手信息** | 独家采访、内部爆料、原创调研数据 | 晚点 LatePost 的深度报道 |
| **技术深度** | 解释了"为什么"和"怎么做"，而不只是"发生了什么" | 架构分析、源码解读、工程复盘 |
| **决策价值** | 读完后能直接影响你的技术选型或商业判断 | "为什么我们从 Kafka 迁移到 Pulsar" |
| **稀缺性** | 这个信息点在其他主流媒体上很难找到 | 小众但精准的行业内幕 |

### 打 `0` 的条件（满足任意一条即可）

| 条件 | 说明 | 示例 |
|------|------|------|
| **纯转载/聚合** | 没有原创内容，只是拼凑其他媒体的报道 | "据XX报道，据YY称..." |
| **企业公关稿** | 明显是公司发布的产品宣传或融资公告 | "XX公司今日宣布完成A轮融资..." |
| **财报数据罗列** | 只有数字堆砌，没有分析 | "Q3营收同比增长23%，净利润..." |
| **标题党/情绪化** | 标题夸张，正文内容撑不起标题 | "震惊！这家公司做了一件..." |
| **过时信息** | 超过 30 天的旧新闻被重新包装发布 | |
| **广告软文** | 正文中出现品牌植入或明显引流行为 | "点击文末链接领取优惠..." |

## 边界案例处理

**不确定时怎么办？**

> 问自己这个问题：**"我会把这篇文章的链接发给一个我尊重的工程师朋友吗？"**
> - 会 → `1`
> - 不会或犹豫 → `0`

**中立的新闻快讯（不好也不坏）**
- 纯事实陈述，没有分析，字数 < 500 → `0`
- 纯事实陈述，但是重大事件首发（如新产品发布、重要政策落地）→ `1`

**财报分析 vs 财报数据**
- 只有数字 → `0`
- 数字 + 作者对业务趋势的判断 → `1`

---

## 标注效率建议

1. **不要逐字阅读正文**，先看标题 + 来源 + 前 200 字，3 秒内做判断
2. 遇到真正拿不准的，标 `?`，最后统一讨论
3. 每标注 50 篇休息一下，避免标注疲劳导致标准漂移
4. 目标：**300 篇在 2-3 小时内完成**

---

## 标注完成后

将标注好的 CSV 文件保存，然后运行评测脚本：

```bash
python scripts/eval_classifier.py --input ground_truth_samples.csv
```

这个脚本会把你的人工标注作为 Ground Truth，测试大模型分类器的 F1 分数。
